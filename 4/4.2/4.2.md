# 4.2. Diccionario de Datos

# Entidad: Empleado

**Semántica:** Representa a los trabajadores de la empresa que están autorizados para realizar operaciones dentro del sistema. 

| ATRIBUTO   | NATURALEZA | FORMATO | VALORES VÁLIDOS     | UNIDAD | DERIVADA DE | SEMÁNTICA             | ONTOLOGÍA                                                                 |
|------------|------------|---------|---------------------|--------|-------------|-----------------------|---------------------------------------------------------------------------|
| idEmpleado | CHAR       | X999    | 1 letra + 3 dígitos | -      | -           | Identificador único   | Identifica de manera única al empleado como agente responsable del registro de seguimientos. Permite establecer trazabilidad de las acciones realizadas por personal autorizado. |
| Nombre     | CHAR       | X(64)   | NOT NULL            | -      | -           | Nombre completo       | Permite conocer el nombre del empleado, lo que facilita la identificación humana del agente asociado al seguimiento. |
| Cargo      | CHAR       | X(64)   | NOT NULL            | -      | -           | Puesto/función        | Describe el rol funcional del empleado dentro de la organización, lo cual es relevante para delimitar responsabilidades. |


# Entidad: Paquete

**Semántica:** representa los objetos físicos o informacionales que son enviados por la empresa a los clientes. Esto es, el pedido del cliente.


| ATRIBUTO        | NATURALEZA | FORMATO  | VALORES VÁLIDOS     | UNIDAD | DERIVADA DE | SEMÁNTICA              | ONTOLOGÍA                                                                 |
|-----------------|------------|----------|---------------------|--------|-------------|------------------------|---------------------------------------------------------------------------|
| idPaquete       | CHAR       | X999     | 1 letra + 3 dígitos | -      | -           | Identificador único    | Identifica de manera única al paquete como objeto físico o informacional dentro del sistema logístico. |
| NumPedido       | CHAR       | X999     | 1 letra + 3 dígitos | -      | -           | Número de pedido       | Establece una relación transaccional con el pedido original. Permite rastrear el origen comercial del paquete. |
| FechaRegistro   | DATE       | AAAAMMDD | Fecha válida        | días   | -           | Fecha de registro      | Indica el momento en que el paquete fue registrado en el sistema. Permite calcular el tiempo transcurrido desde su creación. |
| EstadoActual    | CHAR       | X(64)    | NOT NULL            | -      | -           | Estado del paquete     | Informa sobre la situación actual del paquete (ej. "en tránsito", "entregado"). Es clave para el seguimiento en tiempo real. |
| IdCliente       | CHAR       | X999     | 1 letra + 3 dígitos | -      | Cliente     | Cliente asociado       | Establece la relación de pertenencia entre el paquete y su cliente correspondiente. Permite asociar el envío con el receptor final. |

# Entidad: Seguimiento

**Semántica:** permite registrar los eventos relacionados con la consulta o verificación del estado de los paquetes. 

| ATRIBUTO       | NATURALEZA | FORMATO  | VALORES VÁLIDOS     | UNIDAD | DERIVADA DE | SEMÁNTICA                 | ONTOLOGÍA                                                                 |
|----------------|------------|----------|---------------------|--------|-------------|---------------------------|---------------------------------------------------------------------------|
| IdSeguimiento  | CHAR       | X999     | 1 letra + 3 dígitos | -      | -           | Identificador único       | Identifica de forma única cada evento de seguimiento, permitiendo registrar múltiples acciones sobre un mismo paquete. |
| FechaConsulta  | DATE       | AAAAMMDD | Fecha válida        | días   | -           | Fecha de consulta         | Indica el momento en que se realizó una verificación del estado del paquete. Ayuda a construir una línea temporal de eventos. |
| Medio          | CHAR       | X(64)    | NOT NULL            | -      | -           | Medio de contacto         | Especifica cómo se realizó el seguimiento (ej. vía telefónica, web). Informa sobre la interacción usada. |
| idPaquete      | CHAR       | X999     | 1 letra + 3 dígitos | -      | Paquete     | Paquete asociado          | Relaciona el seguimiento con un paquete específico. Establece el objeto de la acción registrada. |
| idEmpleado     | CHAR       | X999     | 1 letra + 3 dígitos | -      | Empleado    | Empleado registrador      | Asocia el seguimiento a la persona que lo realizó, permitiendo establecer responsabilidad operativa. |

---
# Entidad: Cliente
 
**Semántica:** representa a las personas que solicitan o reciben paquetes a través del sistema.

| ATRIBUTO   | NATURALEZA | FORMATO | VALORES VÁLIDOS     | UNIDAD | DERIVADA DE | SEMÁNTICA             | ONTOLOGÍA                                                                 |
|------------|------------|---------|---------------------|--------|-------------|-----------------------|---------------------------------------------------------------------------|
| idCliente  | CHAR       | X999    | 1 letra + 3 dígitos | -      | -           | Identificador único   | Identifica de forma única a cada cliente dentro del sistema. Establece una relación directa con los paquetes que posee. |
| Nombre     | CHAR       | X(64)   | NOT NULL            | -      | -           | Nombre completo       | Permite conocer el nombre del cliente, facilitando su identificación como sujeto de la relación comercial. |
| Teléfono   | CHAR       | X(15)   | Formato válido      | -      | -           | Contacto telefónico   | Indica un canal de comunicación directo con el cliente. Es útil para gestionar notificaciones o atención personalizada. |
| Correo     | CHAR       | X(64)   | Email válido        | -      | -           | Correo electrónico    | Especifica una vía de contacto digital para enviar información relacionada con pedidos o seguimientos. |

---

# Entidad: Vehículo

**Semántica:** representa los medios de transporte utilizados para realizar los traslados de paquetes dentro del sistema logístico.

| ATRIBUTO        | NATURALEZA | FORMATO | VALORES VÁLIDOS     | UNIDAD | DERIVADA DE | SEMÁNTICA                      | ONTOLOGÍA                                                                 |
|-----------------|------------|---------|---------------------|--------|-------------|--------------------------------|---------------------------------------------------------------------------|
| idVehiculo      | CHAR       | X999    | 1 letra + 3 dígitos | -      | -           | Identificador único            | Identifica de forma única a cada vehículo dentro del sistema logístico. Establece la relación con las cargas asignadas. |
| Placa           | CHAR       | X(10)   | Formato placa válido | -      | -           | Número de matrícula            | Permite identificar legalmente al vehículo y rastrear su uso en las operaciones logísticas. |
| Marca           | CHAR       | X(32)   | NOT NULL             | -      | -           | Marca comercial del vehículo   | Indica el fabricante del vehículo, útil para el mantenimiento, inventario y gestión de flotas. |
| Modelo          | CHAR       | X(32)   | NOT NULL             | -      | -           | Modelo específico               | Permite registrar las especificaciones particulares del vehículo para planificación y control operativo. |
| AñoFabricación  | NUM        | 9999    | 1900-Actualidad      | Año    | -           | Año de fabricación             | Indica el año en que fue fabricado el vehículo, relevante para gestión de antigüedad y depreciación. |
| CapacidadCarga  | NUM        | 99999   | >0                   | Kg     | -           | Capacidad máxima de carga      | Especifica la carga máxima que puede transportar el vehículo, fundamental para planificación de rutas y seguridad. |
