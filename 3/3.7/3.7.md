### 3.7.1 REQUERIMIENTOS DEL MÓDULO

#### 3.7.1.1 REQUERIMIENTOS FUNCIONALES

- **RF01**: Registro de orden de trabajo.
- **RF02**: Aprobación de orden de trabajo.	
- **RF03**: Segmentación de requerimientos.	
- **RF04**: Distribución de requerimientos	
- **RF05**: Asignación de vehículo y conductor	
- **RF06**: Generación de guía de remisión	
- **RF07**: Seguimiento del proceso de picking y precintado	
- **RF08**: Verificación de entrega	
- **RF09**: Evaluación de calidad del servicio	
- **RF10**: Generación de informes de orden de trabajo	
- **RF11**: Notificación a actores involucrados
  
#### 3.7.1.1 REQUERIMIENTOS NO FUNCIONALES

- **RF01**: Disponibilidad
- **RF02**: Tiempo de respuesta	
- **RF03**: Seguridad de la información		
- **RF04**: Escalabilidad	
- **RF05**: Usabilidad
- **RF06**: Compatibilidad
- **RF07**: Mantenibilidad	
- **RF08**: Respaldo de datos	
- **RF09**: Integridad de los datos		

### Caso de Uso 1: Aprobar orden de trabajo

| **Campo**            | **Descripción**                                                                                                |
|----------------------|----------------------------------------------------------------------------------------------------------------|
| **Objetivo**         | Validar la orden de trabajo para continuar con su ejecución.                                                   |
| **Descripción**      | El jefe de planificación revisa la orden generada y aprueba si cumple con los criterios técnicos y logísticos. |
| **Actor Primario**   | Jefe de planificación.                                                                                         |
| **Actor Secundario** | Sistema de planificación                                                                                       |
| **Precondiciones**   | La orden de trabajo ha sido registrada correctamente.                                                          |
| **Flujo de Eventos** | 1. El sistema notifica una nueva orden.<br>2. EL jefe accede al sistema.<br>3. Verifica los detalles técnicos.<br>4. Aprueba o rechaza la orden |
| **Postcondición**    | La orden es marcada como aprobada y pasa a la siguiente etapa.                                                 |

---

### Caso de Uso 2: Asignar vehículo y conductor

| **Campo**            | **Descripción**                                                                                    |
|----------------------|----------------------------------------------------------------------------------------------------|
| **Objetivo**         | Asegurar que el requerimiento tenga el recurso de transporte asignado.                             |
| **Descripción**      | El coordinador de transporte evalúa el requerimiento y asigna un vehículo y conductor disponible.  |
| **Actor Primario**   | Coordinador de transporte.                                                                         |
| **Actor Secundario** | Conductores y sistema de gestión de flota.                                                         |
| **Precondiciones**   | Se debe haber validado el requerimiento de transporte.                                             |
| **Flujo de Eventos** | 1. El sistema notifica un requerimiento aprobado.<br>2. El coordinador revisa los recursos disponibles.<br>3. Asigna vehículo y conductor.<br>4. El sistema actualiza la orden con los datos del transporte. |
| **Postcondición**    | El transporte queda listo para su ejecución.                                                       |

---

### Caso de Uso 3: Realizar picking y precintado


| **Campo**            | **Descripción**                                                         |
|----------------------|-------------------------------------------------------------------------|
| **Objetivo**         | Preparar correctamente el pedido para su despacho.                      |
| **Descripción**      | El personal de almacén recoge los productos, los embala y los precinta. |
| **Actor Primario**   | Personal de almacén.                                                    |
| **Actor Secundario** | Sistema de inventario.                                                  |
| **Precondiciones**   | La orden de trabajo debe estar aprobada y la guía de remisión emitida.  |
| **Flujo de Eventos** | 1. Se imprime la orden de picking.<br>2. El personal recoge los productos.<br>3. Verficica cantidades.<br>4. Embala y precinta.<br>5. Marca la orden como lista para despacho. |
| **Postcondición**    | Productos preparados y validados para entrega.                          |

---

### Caso de Uso 4: Recepcionar orden de trabajo


| **Campo**            | **Descripción**                                                                                                                                                               |
|----------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Objetivo**         | Registrar el inicio del proceso con base en una solicitud del cliente.                                                                                                        |
| **Descripción**      | Se recibe la solicitud de orden de trabajo por parte del cliente a través de un canal habilitado (web, teléfono o presencial) y se registra en el sistema para su validación. |
| **Actor Primario**   | Cliente / Usuario solicitante.                                                                                                                                                |
| **Actor Secundario** | Área de planificación.                                                                                                                                                        |
| **Precondiciones**   | Deben estar habilitados los canales de recepción y el sistema de registro debe estar operativo.                                                                               |
| **Flujo de Eventos** | 1. El cliente solicita una orden de trabajo.<br>2. El área de planificación recibe la solicitud.<br>3. Se verifica la información básica (tipo de requerimiento, fecha, ubicación, etc).<br>4. Se registra la orden en el sistema. |
| **Postcondición**    | La orden queda registrada y lista para ser evaluada y aprobada.                                                                                                               |

---

### Caso de Uso 5: Distribuir requerimientos


| **Campo**            | **Descripción**                                                                                                                                            |
|----------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Objetivo**         | Organizar internamente la ejecución de requerimientos según especialidad o área responsable.                                                               |
| **Descripción**      | Una vez segmentados los requerimientos, se distribuyen a las áreas correspondientes (almacén, transporte, calidad, etc.) para su preparación y ejecución.  |
| **Actor Primario**   | Coordinador de almacén.                                                                                                                                    |
| **Actor Secundario** | Personal operativo (almacén, transporte, calidad).                                                                                                         |
| **Precondiciones**   | El requerimiento ya debe haber sido segmentado y validado.                                                                                                 |
| **Flujo de Eventos** | 1. El coordinador accede al listado de requerimientos segmentados.<br>2. Asigna cada requerimiento al área correspondiente.<br>3. El sistema notifica a cada responsable. |
| **Postcondición**    | Los requerimientos quedan en manos de los responsables de su ejecución.                                                                                    |

---

### Caso de Uso 6: Verificar entrega al cliente


| **Campo**            | **Descripción**                                                                                                                       |
|----------------------|---------------------------------------------------------------------------------------------------------------------------------------|
| **Objetivo**         | Confirmar que los productos y/o servicios han sido entregados al cliente final.                                                       |
| **Descripción**      | El conductor realiza la entrega en el punto de destino y solicita firma o conformidad del cliente, lo que se registra en el sistema.  |
| **Actor Primario**   | Conductor.                                                                                                                            |
| **Actor Secundario** | Cliente receptor.                                                                                                                     |
| **Precondiciones**   | La carga debe haber salido del almacén con guía de remisión.                                                                          |
| **Flujo de Eventos** | 1. El conductor llega al destino.<br>2. Entrega los productos al cliente.<br>3. Solicita firma o confirmación digital/física.<br>4. Registra el comprobante de entrega. |
| **Postcondición**    | El sistema registra que la entrega fue realizada satisfactoriamente.                                                                  |

---

### Caso de Uso 7: Evaluar calidad del servicio entregado


| **Campo**            | **Descripción**                                                                                                                                                      |
|----------------------|---------------------------------------------------------------------------------|
| **Objetivo**         | Determinar si el proceso de entrega y ejecución del requerimiento cumplió los estándares de calidad establecidos.                                                    |
| **Descripción**      | El área de calidad revisa los indicadores (tiempo de entrega, estado del producto, satisfacción del cliente) y valida si se cumplió con los estándares del servicio. |
| **Actor Primario**   | Responsable de calidad.                                                                                                                                              |
| **Actor Secundario** | Cliente (retroalimentación), sistema de monitoreo.                                                                                                                   |
| **Precondiciones**   | El requerimiento debe haber sido completado y entregado.                                                                                                             |
| **Flujo de Eventos** | 1. El sistema genera una alerta de requerimiento entregado.<br>2. El responsable de calidad accede a los detalles del proceso.<br>3. Revisa tiempos, documentación, observaciones del cliente.<br>4. Emite una evaluación con resultado conforme o no conforme. |
| **Postcondición**    | Se almacena el informe de calidad para futuras auditorías o acciones de mejora.                                                                                      |
