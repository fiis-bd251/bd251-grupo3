## **Requerimientos funcionales**


---

### Caso de Uso 1: Registrar ingreso de productos al almacén

| **Campo**            | **Descripción**                                                                 |
|----------------------|---------------------------------------------------------------------------------|
| **Objetivo**         | Permitir que el responsable de almacén registre el ingreso de productos para su almacenamiento y control. |
| **Descripción**      | Proceso mediante el cual se registra la llegada de productos al almacén.       |
| **Actor Primario**   | Responsable de almacén                                                         |
| **Actor Secundario** | Sistema de gestión de almacenes                                                |
| **Precondiciones**   | Los productos deben haber sido recibidos físicamente.                          |
| **Flujo de Eventos** | 1. El responsable accede al sistema.<br>2. Selecciona "Registrar ingreso".<br>3. Ingresa los datos del producto (código, nombre, cantidad, proveedor, fecha).<br>4. El sistema verifica y guarda la información. |
| **Postcondición**    | Los productos quedan registrados como disponibles en el almacén.              |

### Caso de Uso 7: Crear alertas de stock mínimo

| **Campo**            | **Descripción**                                                                 |
|----------------------|---------------------------------------------------------------------------------|
| **Objetivo**         | Configurar alertas cuando el stock de un producto baje de un nivel mínimo.      |
| **Descripción**      | Ayuda a prevenir quiebres de stock mediante alertas automáticas.                |
| **Actor Primario**   | Responsable de almacén                                                         |
| **Actor Secundario** | Sistema de gestión de almacenes                                                |
| **Precondiciones**   | El producto debe estar registrado y tener un límite mínimo configurado.         |
| **Flujo de Eventos** | 1. Accede al sistema.<br>2. Selecciona producto.<br>3. Configura nivel mínimo.<br>4. El sistema activa la alerta cuando se cumple la condición. |
| **Excepciones**      | Si no hay cantidad mínima configurada, no se genera alerta.                    |
| **Postcondición**    | El sistema notifica cuando el producto baja del nivel mínimo.                 |
