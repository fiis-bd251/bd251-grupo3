# 3.1. Módulo 1

# Caso de Uso 1: Generar manifiesto de carga

| Campo             | Descripción                                                                 |
|-------------------|-----------------------------------------------------------------------------|
| **Objetivo**      | Permitir al responsable generar un manifiesto que consolide paquetes para envío. |
| **Descripción**   | Proceso para agrupar paquetes por destino, asignar transporte y emitir el manifiesto. |
| **Actor Primario**| Coordinador de manifiestos                                                  |
| **Actor Secundario** | Sistema de gestión logística                                             |
| **Precondiciones**| Los paquetes deben estar verificados y listos para despacho.                |
| **Flujo de Eventos** | 1. Accede al sistema de gestión logística.  <br>2. Selecciona la opción "Generar manifiesto".  <br>3. Filtra y agrupa los paquetes por destino o ruta.  <br>4. Asigna el vehículo y conductor disponibles.  <br>5. Revisa los datos del manifiesto. <br>6. Confirma y genera el manifiesto en PDF/digital.  <br>7. El sistema notifica al área de despacho y seguimiento. |
| **Postcondición** | El manifiesto queda registrado y disponible para impresión o envío digital. |


# Caso de Uso 2: Consultar manifiestos generados

| Campo             | Descripción                                                                 |
|-------------------|-----------------------------------------------------------------------------|
| **Objetivo**      | Permitir al usuario consultar manifiestos previamente generados.            |
| **Descripción**   | Proceso para acceder al historial de manifiestos, revisar detalles y descargar documentos. |
| **Actor Primario**| Coordinador de manifiestos                                                  |
| **Actor Secundario** | Sistema de gestión logística                                             |
| **Precondiciones**| Debe existir al menos un manifiesto generado previamente.                   |
| **Flujo de Eventos**  |1. Accede al sistema.  <br>2. Selecciona la opción "Consultar manifiestos". <br>3. Ingresa filtros como fecha, destino, transportista o número de manifiesto.  <br>4. El sistema muestra los resultados. <br>5. Selecciona un manifiesto para ver sus detalles.  <br>6. Descarga el documento si lo requiere.
| **Postcondición** | El usuario visualiza o descarga información del manifiesto consultado.      |

# Caso de Uso 3: Anular manifiesto de carga

| Campo             | Descripción                                                                 |
|-------------------|-----------------------------------------------------------------------------|
| **Objetivo**      | Permitir anular un manifiesto generado por error o cambios logísticos.      |
| **Descripción**   | Proceso para invalidar un manifiesto y liberar paquetes asociados.          |
| **Actor Primario**| Coordinador de manifiestos                                                  |
| **Actor Secundario** | Sistema de gestión logística                                             |
| **Precondiciones**| El manifiesto debe estar en estado "pendiente" y no haber sido despachado.  |
| **Flujo de Eventos** |1. Accede al sistema.  <br>2. Selecciona "Anular manifiesto".  <br>3. Busca el manifiesto por número o filtros.  <br>4. Confirma la anulación.  <br>5. El sistema libera los paquetes para reasignación.  <br>6. Se genera un historial de la anulación.|
| **Postcondición** | El manifiesto queda anulado y los paquetes pueden ser reasignados.          |

# Caso de Uso 4: Editar información antes de confirmar manifiesto

| Campo             | Descripción                                                                 |
|-------------------|-----------------------------------------------------------------------------|
| **Objetivo**      | Permitir ajustar información del manifiesto antes de su confirmación final. |
| **Descripción**   | Proceso para editar destino, unidad, chofer o lista de paquetes antes del envío. |
| **Actor Primario**| Coordinador de manifiestos                                                  |
| **Actor Secundario** | Sistema de gestión logística                                             |
| **Precondiciones**| El manifiesto debe estar en estado "en edición" o "pendiente".              |
| **Flujo de Eventos** |1. Accede al sistema.  <br>2. Busca el manifiesto en estado editable.  <br>3. Modifica los datos requeridos (vehículo, ruta, paquetes).  <br>4. Revisa los cambios.  <br>5. Guarda la edición.  <br>6. Continúa el flujo hacia confirmación de manifiesto.|
| **Postcondición** | La información del manifiesto queda actualizada antes de su validación final. |
